# Тестирование на Android-устройстве

Инструкция для проверки приложения **recordsbl** на одном телефоне с Android (без iPhone).

## 1. Подготовка телефона

1. **Включите режим разработчика**: Настройки → О телефоне → 7 раз нажмите «Номер сборки».
2. **Включите отладку по USB**: Настройки → Для разработчиков → «Отладка по USB».
3. Подключите USB, на запрос «Разрешить отладку?» согласитесь (при необходимости отметьте «Всегда с этого компьютера»).
4. На компьютере установите [Android Studio](https://developer.android.com/studio) или только [platform-tools](https://developer.android.com/tools/releases/platform-tools) (команда `adb`).

### macOS / zsh: `command not found: adb`

`adb` не попадает в `PATH` автоматически. Сделайте так:

1. **Убедитесь, что установлены Platform-Tools**  
   Android Studio → **Settings** (или **Android Studio** → **Settings** на macOS) → **Languages & Frameworks** → **Android SDK** → вкладка **SDK Tools** → включите **Android SDK Platform-Tools** → **Apply**.

2. **Путь к SDK** на Mac обычно такой:  
   `~/Library/Android/sdk`  
   (тот же путь показан вверху окна **Android SDK** в настройках.)

3. **Добавьте `platform-tools` в PATH** (один раз), в терминале:

   ```bash
   echo 'export ANDROID_HOME="$HOME/Library/Android/sdk"' >> ~/.zshrc
   echo 'export PATH="$PATH:$ANDROID_HOME/platform-tools"' >> ~/.zshrc
   source ~/.zshrc
   ```

   Если SDK у вас в другом каталоге — подставьте его в `ANDROID_HOME`.

4. Проверка:

   ```bash
   which adb
   adb version
   adb devices
   ```

Временно без правки `~/.zshrc` можно вызвать полный путь:

```bash
~/Library/Android/sdk/platform-tools/adb devices
```

Проверка подключения:

```bash
adb devices
```

Должен отображаться ваш серийный номер и статус `device`.

### macOS / zsh: `command not found: flutter`

Android Studio **не** добавляет Flutter в систему. Нужен отдельный [Flutter SDK](https://docs.flutter.dev/get-started/install/macos):

- **Homebrew:** `brew install --cask flutter` — затем откройте новое окно терминала и проверьте `which flutter`.
- **Вручную:** распакуйте SDK (например в `~/development/flutter`) и добавьте в `~/.zshrc`:

  ```bash
  echo 'export PATH="$PATH:$HOME/development/flutter/bin"' >> ~/.zshrc
  source ~/.zshrc
  ```

  Путь должен указывать на каталог **`bin`**, где лежит исполняемый файл `flutter`.

Проверка: `flutter --version` и `flutter doctor`.

### macOS: Gradle / `Too many open files`

При сборке (`assembleDebug`) Gradle может открыть много файлов и сокетов. Если в логе **`Too many open files`**, увеличьте лимит в **том же терминале**, где запускаете Flutter (действует до закрытия окна):

```bash
ulimit -n 10240
flutter run
```

Чтобы сделать это постоянным для интерактивных сессий zsh:

```bash
echo 'ulimit -n 10240' >> ~/.zshrc
source ~/.zshrc
```

(При необходимости поднимите значение; на некоторых системах максимум задаётся в `/etc/sysctl.conf` или через `launchctl` — для обычной разработки `10240` обычно хватает.)

### Иконка приложения в лаунчере (список приложений)

Чтобы на экране телефона была **ваша** картинка, а не стандартный значок Flutter, после изменений в `assets/branding/aist_logo.png` выполните:

```bash
cd mobile && dart pub get && dart run flutter_launcher_icons
```

Затем пересоберите и переустановите приложение (`flutter run`). Иконка в шапке приложения берётся из того же файла через `Image.asset`.

### Логотип `aist_logo.png` и ошибка «The file name must end with .xml»

PNG **нельзя** класть в произвольную папку внутри `android/.../res/` (например `res/img/`): Gradle ожидает только стандартные типы ресурсов (`drawable`, `mipmap`, `xml` и т.д.). Логотип для Flutter лежит в **`mobile/assets/branding/aist_logo.png`** и подключён в `pubspec.yaml`. Скопируйте туда файл вручную, если его ещё нет.

### Сборка падала на плагине `disk_space`

В проекте зависимость **disk_space** убрана: у плагина устаревший Gradle/jcenter и он ломал сборку. Проверка свободного места на Android сделана без этого пакета (см. `lib/services/disk_space_service.dart`).

## 2. Проект Flutter

В каталоге репозитория:

```bash
cd mobile
flutter pub get
```

Если папок `android/` ещё нет:

```bash
flutter create . --project-name recordsbl --org com.company.recordsbl
```

Сборка и установка на подключённое устройство:

```bash
flutter run
```

Или только APK для ручной установки:

```bash
flutter build apk --debug
# файл: build/app/outputs/flutter-apk/app-debug.apk
```

Установка через adb:

```bash
adb install -r build/app/outputs/flutter-apk/app-debug.apk
```

## 3. Доступ к API с телефона

Телефон **не** достучится до `http://127.0.0.1` на компьютере.

Выберите один вариант:

- **Тот же Wi‑Fi**: узнайте IP компьютера (например `192.168.1.10`), на сервере слушайте `0.0.0.0`, в настройках приложения укажите `http://192.168.1.10:3000` (порт как у вашего API).
- **Туннель** (ngrok, cloudflared и т.п.): выдайте HTTPS URL и укажите его в настройках приложения.

Убедитесь, что брандмауэр на ПК не блокирует порт API.

## 4. Разрешения на телефоне

При первом запуске выдайте **микрофон** (запись). Если запись не стартует — Настройки → Приложения → recordsbl → Разрешения → Микрофон.

## 5. Чек-лист проверок (Android)

Отмечайте по мере прохождения.

### Базовое

- [ ] Приложение устанавливается и запускается без краша.
- [ ] В настройках сохраняются **логин** и **URL сервера**; после перезапуска значения на месте.

### Запись

- [ ] «Начать запись встречи» — запись начинается, время идёт, полоска уровня реагирует на звук.
- [ ] **Пауза** / **Продолжить** — без краша, после «Стоп» встреча в списке с правдоподобной длительностью.
- [ ] Выбор **места встречи** до старта отображается в карточке после сохранения.

### Система

- [ ] Входящий **звонок** во время записи: приложение уходит в паузу (или корректно освобождает микрофон); после сброса можно продолжить или остановить.
- [ ] **Сворачивание** приложения во время записи: по ТЗ ожидается непрерывный захват в фоне — проверьте, что запись не обрывается неожиданно (зависит от финальной реализации фона).

### Диск (по возможности)

- [ ] При очень малом свободном месте — предупреждение или блокировка новой записи (как в текущей логике приложения).

### Выгрузка

- [ ] «Выгрузить» при работающем сервере и сети: прогресс, затем статус вроде «Выгружена».
- [ ] **Обрыв Wi‑Fi** во время выгрузки: ошибка, повтор позже или после возврата в приложение.

### Очистка

- [ ] «Удалить» у записи — запись пропадает из списка.

## 6. Полезные команды adb

Просмотр логов приложения (фильтр по тегам Flutter/Dart):

```bash
adb logcat | grep -i flutter
```

Скриншот экрана:

```bash
adb exec-out screencap -p > screen.png
```

Список установленных пакетов (проверка имени пакета):

```bash
adb shell pm list packages | grep recordsbl
```

## 7. Если что-то не работает

| Симптом | Что проверить |
|--------|----------------|
| `adb devices` пустой | Кабель, режим USB (передача файлов), драйверы, другой порт. |
| `INSTALL_FAILED_USER_RESTRICTED` / «Install canceled by user» | Сборка уже успешна; телефон **запретил установку по USB**. На **Xiaomi / Redmi / POCO (MIUI/HyperOS)**: *Настройки → Дополнительно → Для разработчиков* — включить **«Установка через USB»** и при необходимости **«Отладка по USB (настройки безопасности)»**; на экране при установке нажать **Разрешить/Установить**. Повторить `flutter run` или `adb install -r build/app/outputs/flutter-apk/app-debug.apk`. |
| Сборка падает | `flutter doctor`, установленный Android SDK, лицензии `flutter doctor --android-licenses`. |
| Нет сети до API | URL в настройках, IP/туннель, firewall, сервер слушает не только localhost. |
| Нет звука в записи | Разрешение микрофона, не заглушен ли вход, другие приложения не держат микрофон. |
